# Wave 0 — De-risk the Load-bearing Bets (Spike) — Implementation Plan

> **How to execute:** This is a **spike**, not production TDD. Each task is a notebook cell (or two) that ends in a *recorded observation*, not a passing test. Run top to bottom in one Colab/Kaggle session. Steps use checkbox (`- [ ]`) syntax for tracking. Deviation from the standard writing-plans TDD format is intentional and is called out where it matters — spike code is throwaway, but the **scenario fixture** and **findings** it produces graduate into the real repo (evals/fixtures and the ADR log).

**Goal:** On a free notebook GPU, confirm the 4B-class candidates can (1) drive a 2–3 step tool-calling loop into a structured, evidence-grounded diagnosis, and (2) be LoRA fine-tuned on free Colab/Kaggle for their exact architecture — then pick the default reasoner and record it.

**Architecture:** A minimal model-agnostic ReAct loop (JSON action protocol) over two canned tools (`query_logs`, `get_recent_commits`) runs one hand-built bad-deploy incident through each candidate model and a hosted reference (Gemini free tier). A separate fine-tune smoke test runs a tiny SFT on each candidate and confirms it trains, loss drops, and exports. Findings drive the model gate.

**Tech Stack:** Google Colab (free T4 ~15 GB) *or* Kaggle (free T4×2 / P100 16 GB) · Unsloth · Transformers v5 / latest · PyTorch · `google-generativeai` (Gemini free tier reference) · Python 3.

**Candidates (from DR-0007):**
- **Qwen3.5-4B** — `unsloth/Qwen3.5-4B` (newest; GDN+VLM; bf16 LoRA; official free Colab notebook).
- **Qwen3-4B** — `unsloth/Qwen3-4B` (dense; mature 4-bit QLoRA; text-only; fallback).
- *(bigger free-trainable reference, only if 4B is weak)* **Qwen3-8B** — `unsloth/Qwen3-8B`.

## Task 3: Canned tools + a minimal model-agnostic ReAct loop (capability spike)

**Cell(s):** define two tools and one legible loop. The loop is JSON-action ReAct so it works identically across model families — this also foreshadows the real Wave 1 agent loop (priority #2, orchestration).

- [ ] **Step 1: Define the two canned tools.**

```python
def query_logs(level=None, since=None):
    rows = FIXTURE["logs"]
    if level: rows = [r for r in rows if r["level"] == level]
    if since: rows = [r for r in rows if r["ts"] >= since]
    return rows

def get_recent_commits(n=5):
    return FIXTURE["commits"][:n]

TOOLS = {"query_logs": query_logs, "get_recent_commits": get_recent_commits}
```

- [ ] **Step 2: Write the system prompt (the loop contract).**

```python
SYSTEM = """You are an incident-triage agent. Diagnose the production incident using ONLY evidence
returned by tools. On each turn, output EXACTLY ONE JSON object and nothing else.

To call a tool:
  {"action": "query_logs", "args": {"level": "ERROR"}}
  {"action": "get_recent_commits", "args": {"n": 5}}

When you have enough evidence, output your final answer:
  {"action": "final",
   "insufficient_evidence": false,
   "hypotheses": [
     {"cause": "<one sentence>", "confidence": 0.0-1.0,
      "evidence": ["log:<ts> <route> <status>", "commit:<sha> <file>"]}
   ]}

Rules: rank hypotheses most-likely first. Every evidence string MUST correspond to a real row a
tool returned. If the tools do not support a cause, set "insufficient_evidence": true and do not invent one.
"""
```

- [ ] **Step 3: JSON extractor (robust to a `<think>` block).** Qwen3.5/Qwen3 default to thinking mode, so the model may emit `<think>...</think>` before the JSON.

```python
import re, json
def extract_json(text):
    text = re.sub(r"<think>.*?</think>", "", text, flags=re.DOTALL)  # strip reasoning
    matches = re.findall(r"\{.*\}", text, flags=re.DOTALL)            # last balanced-ish object
    for m in reversed(matches):
        try: return json.loads(m)
        except json.JSONDecodeError: continue
    return None
```

- [ ] **Step 4: A `chat()` helper (works for any loaded Unsloth model).**

```python
def chat(model, tokenizer, messages, max_new_tokens=512):
    inputs = tokenizer.apply_chat_template(
        messages, tokenize=True, add_generation_prompt=True, return_tensors="pt"
    ).to(model.device)
    out = model.generate(inputs, max_new_tokens=max_new_tokens, do_sample=False,
                         pad_token_id=tokenizer.eos_token_id)
    return tokenizer.decode(out[0][inputs.shape[-1]:], skip_special_tokens=True)
```

- [ ] **Step 5: The ReAct loop.**

```python
def run_loop(model, tokenizer, max_steps=4, verbose=True):
    messages = [{"role": "system", "content": SYSTEM},
                {"role": "user", "content": f"Incident at {FIXTURE['now']}. /login is failing. Diagnose."}]
    trace = []
    for step in range(max_steps):
        raw = chat(model, tokenizer, messages)
        obj = extract_json(raw)
        if verbose: print(f"--- step {step} ---\n{obj}\n")
        if obj is None:
            trace.append(("parse_error", raw)); break
        if obj.get("action") == "final":
            trace.append(("final", obj)); return obj, trace
        tool = TOOLS.get(obj.get("action"))
        if not tool:
            trace.append(("bad_tool", obj)); break
        result = tool(**obj.get("args", {}))
        trace.append((obj["action"], result))
        messages.append({"role": "assistant", "content": json.dumps(obj)})
        messages.append({"role": "user", "content": f"TOOL RESULT:\n{json.dumps(result)}"})
    return None, trace
```

### 6b — Fine-tune smoke test (the real load-bearing bet)

Tiny SFT to prove the *training pipeline runs on a free T4* for each candidate's architecture. Not about quality — about "does it train, does loss drop, does it export."

- [ ] **Step 1: Build a tiny SFT dataset (~12 examples, incident → gold diagnosis).**

```python
from datasets import Dataset
def to_text(tok, logs, commits, gold):
    msgs = [{"role": "system", "content": SYSTEM},
            {"role": "user", "content": f"LOGS:\n{json.dumps(logs)}\nCOMMITS:\n{json.dumps(commits)}"},
            {"role": "assistant", "content": json.dumps(gold)}]
    return tok.apply_chat_template(msgs, tokenize=False)

gold_answer = {"action": "final", "insufficient_evidence": False,
    "hypotheses": [{"cause": FIXTURE["gold_cause"], "confidence": 0.9, "evidence": FIXTURE["gold_evidence"]}]}
# 12 light variants (vary timestamps/sha/module) so the smoke test has >1 row; real data comes in Wave 4.
rows = [{"logs": FIXTURE["logs"], "commits": FIXTURE["commits"], "gold": gold_answer} for _ in range(12)]
```

- [ ] **Step 2: Smoke-train Qwen3-4B (4-bit QLoRA — the clean path).**

```python
from unsloth import FastLanguageModel
from trl import SFTTrainer, SFTConfig
m, t = FastLanguageModel.from_pretrained("unsloth/Qwen3-4B", max_seq_length=2048, load_in_4bit=True)
m = FastLanguageModel.get_peft_model(
    m, r=16, lora_alpha=16, lora_dropout=0, bias="none",
    target_modules=["q_proj","k_proj","v_proj","o_proj","gate_proj","up_proj","down_proj"],
    use_gradient_checkpointing="unsloth", random_state=3407)
ds = Dataset.from_list([{"text": to_text(t, r["logs"], r["commits"], r["gold"])} for r in rows])
trainer = SFTTrainer(model=m, tokenizer=t, train_dataset=ds,
    args=SFTConfig(max_seq_length=2048, per_device_train_batch_size=1,
        gradient_accumulation_steps=4, warmup_steps=2, max_steps=30,
        learning_rate=2e-4, logging_steps=1, output_dir="ft_qwen3_4b", report_to="none"))
print(trainer.train())                  # RECORD: loss trend + seconds/step
m.save_pretrained_gguf("ft_qwen3_4b_gguf", t)   # RECORD: export succeeds
del m, t; gc.collect(); torch.cuda.empty_cache()
```
